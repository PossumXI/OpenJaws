## 2. Security Hardening Gaps

1. **Workspace Trust Boundary**
   - The `showSetupScreens` function (see `src/interactiveHelpers.tsx`) displays a Trust Dialog for interactive sessions. Ensure that any future automation paths (e.g., CI/CD) that skip the dialog still enforce strict validation of external includes (`OpenJawsMdExternalIncludesDialog`).
   - Recommendation: Add unit tests that verify `checkHasTrustDialogAccepted()` correctly blocks untrusted repository paths.

2. **External Markdown Includes**
   - `OpenJawsMdExternalIncludesDialog` warns users about external includes. However, the actual content is fetched via `getMemoryFiles(true)`. Verify that the fetch layer sanitizes URLs and respects same‑origin policies.

3. **Environment Variable Injection**
   - `applyConfigEnvironmentVariables()` runs after the trust dialog but may still ingest untrusted environment variables (e.g., `process.env`). Ensure that any variable used for auth headers or API keys is validated against a whitelist before being used.
   - Recommendation: Add a runtime guard that drops any env var that does not match a known pattern (`^[A-Z0-9_]+$`).

4. **Rate‑Limit Listener**
   - The listener in `src/cli/print.ts` emits `rate_limit_event` for *all* status changes, including `allowed`. Make sure the downstream consumer does not expose internal rate‑limit counters to untrusted UI components.
