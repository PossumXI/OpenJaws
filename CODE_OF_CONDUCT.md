# Code of Conduct

## Our Standard

Contributors are expected to be direct, respectful, and technically honest.

## Expected Behavior

- discuss code and design decisions in concrete terms
- assume good intent while holding a high technical bar
- document risks clearly instead of obscuring them
- keep reviews focused on behavior, correctness, and user impact

## Unacceptable Behavior

- harassment or personal attacks
- doxxing or publishing private information
- intentionally introducing secrets, malware, or unsafe code paths
- misrepresenting benchmark, security, or verification results

## Enforcement

Project maintainers may remove contributions or restrict participation when behavior damages the project or its contributors.
